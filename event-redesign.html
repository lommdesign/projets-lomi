Picky

Event Application Flow Redesign

Picky connects UGC creators with K-beauty brands through events. I redesigned the event application flow, increasing completions by 33%.

Company
Picky
Role
Product designer, end-to-end
Timeline
6 weeks
Team
1 PM, 1 Engineer
Responsibilities
Research, data analysis, UI/UX design

[ The Story ]

Picky connects UGC creators with K-beauty brands through events — creators apply, brands pick, and collaborations happen. It's a simple idea, but the application flow wasn't keeping up.

Most users who started an event application never finished it. I kept seeing the same complaints from users, so I dug into Statsig to understand what was going on. The funnel told a clear story: people were dropping off, and two things were causing it.

[ What was going wrong ]

To apply for events, creators need to fill in profile details like their address, social media, and creator info. I noticed this through user complaints and confirmed it in Statsig — the funnel showed a steep drop-off during the application flow. I dug in and found two root causes:

First, when required info was missing, the app sent users to a separate "My Page" to fill it in. Most never came back. Second, error messages were vague — users didn't know what was wrong or how to fix it.

[ The Goal ]

Increase the event application completion rate by removing friction from the flow.

[ The Approach ]

I explored a few directions before landing on a solution.

The first option was to improve the existing redirect. Keep sending users to My Page, but make the experience better there. It was the smallest change, but it didn't solve the core problem: users were still leaving the event page.

The second was a full-screen modal that collected everything at once. This kept users in context, but it felt heavy, especially for creators who only had one or two fields missing.

The third was a bottom sheet that asked only for what was missing. Lightweight, contextual, and it kept users exactly where they were. That's the one we went with.

[ What We Built ]

Keep users on the event page — the redirect to My Page was where most users dropped off. Collect only missing info via a bottom sheet. Less work, less friction. Add a step indicator so users can see progress and what's left. Replace vague errors with field-level messages that tell users exactly what to fix.

I worked closely with our engineer to rewrite the validation logic. Every error message now points to a specific field and tells the user how to fix it. No more guessing.

[ The Result ]

+33% completed event applications, measured in Statsig.

The biggest change came from a simple shift: instead of asking users to go somewhere else to complete their profile, we brought the form to them. Less navigation, less confusion, more completions.
