AI + Design

Reducing the gap between design and development

I'm exploring how AI tools can make designers more independent — from turning Figma designs into working code, to building reusable skills and agents that any designer on the team can use.

[ Claude Code + Figma MCP ]

From Figma to Flutter with a prompt

I wanted to see how close a designer could get to real development output without writing code from scratch. I set up Claude Code with Figma MCP on our production Flutter codebase — the Picky mobile app — and tested whether I could go from a Figma design to a working screen.

🛠

The setup

Installed Claude Code, cloned the Flutter repo, connected Figma MCP. Then pointed Claude at a specific Figma node and asked it to create a self-contained high-fidelity wireframe using the existing design system components and tokens.

💬

The prompt that worked

"I have a new design on Figma [node URL]. Use Figma MCP to understand the design first, and use other Figma-related skills to create a self-contained high-fidelity wireframe that includes components from the design system and design system token updates. Make a fresh page with dummy data and access it from the dev control panel."

💡

What I learned

Plan mode vs. Edit mode matters — use Shift+Tab to switch. You can ask Claude things like "what branch am I on?" or "commit these changes." Start a new branch for each prototype experiment. Write prompts in a note first since it's hard to edit in the terminal. Press R to refresh on the simulator to see changes live.

[ The result ]

A working Flutter screen, built from a Figma design, using the actual production design system — created by a designer. Not pixel-perfect, but close enough to validate ideas, test interactions, and have real conversations with engineers using shared code instead of screenshots.

[ Skills & Agents for Designers ]

Building a design toolkit that talks back

A skill is like a recipe — a set of instructions for a specific task. An agent is your digital buddy that uses those recipes. One agent can mix and match many skills. The goal: optimize the designer's workflow, reduce the gap between design and development, and minimize inefficient communication with engineers.

Agent Skills it has
Research Agent Project Scanner, Research Synthesis, User Research
Design Agent Project Scanner, Wireframing, Prototyping, UX Copy
Design Review Agent Project Scanner, Design Feedback, Accessibility Review

How to use them

You just talk naturally — on Claude chat or Claude Code. The right skill kicks in automatically based on your words.

You say... Skill that triggers
"Review this design for me" Design Feedback
"Is this accessible?" Accessibility Review
"Write the copy for this error state" UX Copy

When to create a new skill

Create one when you find yourself repeating the same type of task. If you always format notes the same way — skill it. Your team has a specific handoff checklist — skill it. You summarize your work with a consistent template — skill it.

Sharing across the team

Skills are just markdown files. Share them across designers and everyone uses the same toolkit. Keep refining by gathering feedback from the team, then editing and polishing the skill files together.

[ My take ]

The designer who can prototype in code, set up their own AI tools, and share those tools with their team — that's where design is heading. These skills and agents aren't replacing the craft. They're removing the friction around it.
